# Dnotitia 인원관리 시스템

임직원 엑셀 원본을 업로드하면 조직도, 인원 조회, 대시보드, 인턴 현황을 자동으로 생성해주는
사내 인원관리 웹앱입니다.

## 시작하기

```bash
npm install
npm run db:seed   # data/employees_source.xlsx 를 SQLite(data/app.db)로 적재
npm run dev        # http://localhost:3000
```

`data/employees_source.xlsx` 는 PII(개인정보)가 포함되어 있어 git에 커밋하지 않습니다
(`.gitignore` 처리됨). 새로 시딩하려면 본인의 엑셀 원본을 해당 경로에 두고
`npm run db:seed [기준일자 YYYY-MM-DD]`를 실행하세요.

## 기능

- **조직도** (`/orgchart`) — 상위조직1/2/부서/직책 구조를 엑셀에서 자동으로 그려줍니다.
  최근 2개월 신규입사/부서이동을 색으로 표시합니다.
- **인원 조회** (`/query`) — 기준일자, 연구원 구분, 나이, 입사연도, 인턴 여부 등 조건으로
  인원 수와 명단을 조회하고 CSV로 내보낼 수 있습니다. 자주 쓰는 조건은 프리셋 버튼으로 제공합니다.
- **대시보드** (`/`) — 본부/그룹/부서별, 분류별, 연령/연차 분포, 연구원 비율 등을 차트로 보여줍니다.
- **인턴 현황** (`/interns`) — 인턴 명단과 계약종료 임박자(30일 이내)를 알려줍니다.
- **데이터 업로드** (`/admin/upload`) — 매월 갱신된 엑셀을 업로드하면 기존 사번은
  최신 정보로 덮어쓰고, 이번 업로드에 없는 사번은 자동으로 퇴사(비활성) 처리됩니다.

## 데이터 모델 & 한계

- 원본 엑셀에는 퇴사일 필드가 없어, "재직 여부"는 입사일 · 계약종료일 · 스냅샷 포함 여부로
  근사 계산합니다. 스냅샷에 포함된 시점(예: 최초 업로드일)에는 계약종료일이 과거여도
  (계약이 갱신된 경우가 있어) 스냅샷 포함 여부를 우선합니다. 최신 스냅샷보다 미래 날짜를
  조회할 때만 계약종료일로 활동 여부를 예측합니다.
- 연차/나이는 매번 생년월일·입사일로부터 계산하며(엑셀의 나이/연차 컬럼은 참고용),
  학위 가산(석사 +2년, 박사 +5년) 등 수기 보정은 반영하지 않습니다.
- 조직도의 "겸직(겸)" 표시는 엑셀에 명시적 필드가 없어 자동 반영하지 않습니다.
